# Local ADRs

This folder holds repository-specific Architecture Decision Records for peripheral firmware.

ADRs here are supplemental and must not override canonical policy, the device model, contracts, or terminology from the Greenhouse Documentation repository.

## Loading Rules

- Read this digest first, then load only the ADRs relevant to the area you are touching — not all of them.
- Keep local ADRs focused on decisions specific to this repository (for example a board-family choice or a peripheral-internal pattern) that do not belong in shared canonical docs.

## ADR Workflow

When adding or updating an ADR, follow the workflow in `AGENTS.md`:

1. Read the central documentation entry point first: https://github.com/thedrewdz/Greenhouse-Documentation/blob/main/README.md
2. Align terms and assumptions with canonical docs before writing.
3. Keep the ADR scoped to a repository-specific decision.
4. On conflict, follow canonical guidance and scope or update the ADR accordingly.
